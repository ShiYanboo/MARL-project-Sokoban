\begin{abstract}
本报告围绕课程组队课题中的``2人合作推箱子''任务，整理了我们在 \code{gym-sokoban} 与 HARL 框架上的环境接入、模型结构、奖励塑形、输入历史、信用分配和算法改动探索。我们将原始双人 Sokoban 封装为 HARL 标准多智能体接口，采用 turn-based 动作执行与 action mask，主线使用 HAPPO 训练，并搭建了 HASAC 与 HAHyPO 等算法入口。实验表明：在固定 7x7、2箱子的设置下，HAPPO 能够明显学习，旧版 BFS 势函数塑形中去除 deadlock 与 pushability 负项的 \code{v1-nodl-nopush} 是最稳定的轻量方案；单纯换 CNN 未带来稳定收益；GRU 与动作历史在早期有一定帮助，但长历史并不稳定；更符合推箱语义的 reverse-push reward shaping v2 在成功率上没有稳定超过旧 BFS 势函数，主要受不可达距离尺度和稀疏团队奖励影响；从 7x7 迁移到 13x13 padding 或真 v0/v1/v2 混合后性能显著下降，说明泛化难度主要来自空间尺度、平移不变性和信用分配，而不是单一超参数。
\end{abstract}

\section{任务与要求}
课程要求在双人 Sokoban 环境中控制两个智能体（环境来自 gym-sokoban~\citep{schrader2018sokoban}），将箱子推到目标位置。作业关注完整训练评估流程、算法设计、创新探索、训练参数、核心代码、实验曲线和结果分析。给定基准包括 HARL 框架~\citep{harlrepo} 中的 HAPPO~\citep{kuba2021trust} 与 HASAC~\citep{zhong2023heterogeneous}；环境原始奖励为团队共享奖励，且多智能体算法需要考虑信用分配。

我们选择 \code{TwoPlayer-Sokoban-v0} 作为主环境，并扩展到 \code{v0/v1/v2} 混合训练。主要目标不是只追求一个最高分策略，而是系统比较以下方向：
\begin{itemize}[leftmargin=1.2em,itemsep=0.2em]
    \item 环境封装：turn-based 执行、action mask、共享状态和日志指标。
    \item 模型结构：MLP、CNN、GRU/RNN 以及动作历史输入。
    \item 奖励塑形：原始奖励、旧 BFS 势函数、reverse-push v2、deadlock/pushability 消融。
    \item 信用分配：共享团队回报、active-agent mask、actor-side credit 辅助项。
    \item 算法探索：HAPPO 主线、HASAC 接入验证、HAHyPO 混合优势函数。
\end{itemize}

\section{系统实现}
\subsection{环境接入与动作执行}
原始双人 Sokoban 环境底层仍是一个 gym 环境。我们在 \path{HARL/harl/envs/sokoban/sokoban_env.py} 中将其封装成 HARL 所需接口：每个 agent 输出一个 \code{Discrete(9)} 动作，分别表示 \code{noop}+四个移动+四个推箱动作。默认采用 turn-based 控制：每步只有当前 active agent 可以选择非 \code{noop} 动作，另一个 agent 的 action mask 中只有 \code{noop} 可用。底层环境奖励复制给两个 agent，critic 使用团队回报。

\begin{lstlisting}[language=Python,caption={Turn-based 动作解析的核心逻辑。}]
def _resolve_action(self, actions):
    chosen_agent = self._priority_agent
    chosen_action = actions[chosen_agent]
    invalid_action_attempt = any(
        actions[i] != 0 for i in range(self.n_agents)
        if i != chosen_agent
    )
    if chosen_action == 0:
        return chosen_agent, 0, info
    if chosen_agent == 0:
        return chosen_agent, chosen_action, info
    return chosen_agent, chosen_action + 8, info
\end{lstlisting}

这种封装有两个好处。第一，它避免两个 agent 在同一步同时改变底层环境导致冲突裁决噪声。第二，inactive agent 的 actor 不会因为不可执行动作被错误训练，后续 credit assignment 也更容易定义。

\subsection{观测与共享状态}
默认观测为符号化向量。每个 agent 看到墙、目标、目标上的箱子、普通箱子、自己位置、队友位置六类二值图，并额外拼接当前 active agent 标志与步数比例。共享状态使用同样的全局符号图。为比较模型结构，我们同时支持 CNN 输入：将上述通道堆叠为 channel-first 图像，并使用三层小型 Sokoban CNN encoder。为研究部分可观测性与轮流执行记忆，环境还支持 \code{action_history_len=K}，将过去 $K$ 步实际执行的联合动作 one-hot 拼接到 observation 或作为常数通道加入 CNN observation。

\subsection{混合地图与 13x13 padding}
历史 \code{scenario_pool=v0,v1,v2} 脚本同时指定了 \code{dim_room=7,num_boxes=2}，因此它们只是``伪混合''：gym scenario 在切换，但地图尺寸和箱子数量没有真正变化。为进行真混合，我们新增固定观测画布：
\begin{itemize}[leftmargin=1.2em,itemsep=0.2em]
    \item \code{use_mixed_obs_padding=True}，\code{obs_dim_room=13}；
    \item padding 模式包括 \code{random_episode}、\code{top_left} 和 \code{center}；
    \item 真 mix 时移除固定 \code{dim_room} 与 \code{num_boxes}，让 \code{v0/v1/v2} 保留原始复杂度差异。
\end{itemize}
这使网络输入维度固定，但显著增加了空间泛化难度。

\section{算法与奖励设计}
\subsection{HAPPO 主线}

\subsection{Reward Shaping}
原始环境奖励为：每步 $-0.1$，箱子推上目标 $+1$，箱子离开目标 $-1$，全部完成 $+10$；部分脚本将完成奖励提升到 $+20$。在此基础上，我们实现了可关闭的势函数型 reward shaping，核心文件为 \path{HARL/harl/envs/sokoban/reward_shaping.py}：
\begin{align}
 r_t =&\ r_t^{base} + w_d(D(s_t)-D(s_{t+1})) \\
      &+ w_a(A(s_t)-A(s_{t+1})) + w_p \min(0,P(s_{t+1})-P(s_t)) \\
      &- \lambda \max(0,L(s_{t+1})-L(s_t)) + w_u \mathbb{1}[\Delta D > 0].
\end{align}
其中 $D$ 是箱子到目标的最小匹配距离，$A$ 是 agent 到箱子或有效推箱站位的距离，$P$ 是可推动方向数，$L$ 是死锁箱数量。我们比较了两套距离语义：
\begin{itemize}[leftmargin=1.2em,itemsep=0.2em]
    \item 旧 BFS：箱子到目标使用墙约束 BFS；agent 到箱子使用相邻格 BFS。典型权重为 \code{distance=0.20, agent_box=0.10}。
    \item v2 reverse-push：箱子到目标使用 reverse-push distance，agent 被引导到能降低全局箱子距离的有效站位，并可用 \code{shaping_unreachable_distance=12} 裁剪不可达距离。
\end{itemize}
最稳定的旧 BFS 方案为 \code{v1-nodl-nopush}：保留 box-target 与 agent-box 正向势函数，去掉 pushability 与 deadlock 负项。

\subsection{信用分配与算法改动}
环境奖励仍是团队共享，但 turn-based 机制使每步只有 active agent 真实影响环境。我们做了两层处理：
\begin{enumerate}[leftmargin=1.4em,itemsep=0.2em]
    \item active mask：inactive agent 只有 \code{noop} 可选，其 actor buffer 中 active mask 为 0。
    \item actor-side credit：critic 仍学习团队 reward，但 actor advantage 可加入 active agent 的小型辅助信用项。
\end{enumerate}

\begin{lstlisting}[language=Python,caption={Actor-side credit 只改变 active actor 的优势函数，不改变 critic 目标。}]
if mode == "active_progress":
    credit = max(base_reward, 0.0)
    credit += max(useful_push_distance_delta, 0.0)
    credit += max(agent_box_distance_shaping_reward, 0.0)
    if action_moved_box:
        credit += 0.2
A_actor = A_team + coef * credit
\end{lstlisting}

此外，我们增加了 HAHyPO 算法入口：保留 HAPPO 的 critic 与 clipped update，但将 actor 优势函数改为 GAE 优势与 rollout 内轨迹回报 z-score 的混合：
\begin{equation}
A^{actor}=(1-\alpha)\,\mathrm{zscore}(A^{GAE})+\alpha\,\mathrm{zscore}(R^{traj}).
\end{equation}
该思路借鉴 group-relative 的优化信号，希望在稀疏团队回报下提供更直接的轨迹级排序信息。当前 HAHyPO 和 actor-credit 更适合作为算法探索，主要可比较结论仍来自 HAPPO 主线曲线。

\section{结果与分析}
\subsection{7x7 固定环境：可以稳定学到协作策略}
图 \ref{fig:old7x7} 展示了 7x7、2 boxes、20M pretrain 加 20M pseudo-mix 的核心曲线。整体趋势说明 HAPPO 在该难度下可以明显脱离随机策略：训练成功率和 box completion ratio 持续上升，critic value loss 在后期下降，eval episode reward 从约 $-15$ 逐步升高到接近或超过 0。

\begin{figure*}[t]
\centering
\includegraphics[width=0.98\textwidth]{figures/fig1_old_7x7_key_metrics.png}
\caption{7x7 固定尺寸环境下的长跑结果。虚线前为 20M v0 pretrain，虚线后为历史 pseudo-mix。\code{v1-nodl-nopush} 的 train success、box completion 和 eval reward 通常最好；GRU/动作历史不是单调更好，8步历史在后期明显偏弱。}
\label{fig:old7x7}
\end{figure*}

在这些曲线中，\code{v1-nodl-nopush} 最稳定：训练成功率后期约在 0.6 左右，eval success 曾出现 0.8 以上峰值，box completion ratio 也最高。原始 baseline 同样能学，但收敛慢、回报较低。GRU 无历史和 1步历史在前期有帮助，但 8步历史并没有进一步提升，说明当前符号状态加 turn flag 已经包含多数决策信息，过长历史反而增加输入维度和非平稳相关性。

\subsection{Reward shaping：语义正确不等于最好学}
图 \ref{fig:v2} 比较了旧 BFS \code{v1-nodl-nopush}、v2 cap12 与 GRU 变体。v2 使用 reverse-push 距离，更接近 Sokoban 中``箱子能否被推到目标''的真实约束；但从 success rate 和 box completion 看，它没有稳定超过旧 BFS。其 reward 曲线明显更高，主要因为 shaped reward 尺度更大，并不能直接解释为成功率更高。

\begin{figure*}[t]
\centering
\includegraphics[width=0.98\textwidth]{figures/fig2_v2_cap12_comparison.png}
\caption{v2 cap12 与旧 BFS shaping 的比较。v2 的 episode reward 被塑形项显著抬高，但 eval success 与 box completion 并未稳定优于旧 BFS；旧 BFS 的 \code{v1-nodl-nopush} 更适合作为主线。}
\label{fig:v2}
\end{figure*}

我们认为原因有三点。第一，reverse-push 距离在不可达状态会产生很大的势函数差，即使用 cap12 裁剪，尺度仍更敏感。第二，Sokoban 的错误推箱会产生长程后果，局部语义更强的 shaping 容易把策略引向短期可达但长期无解的位置。第三，团队共享 reward 下，哪个 agent 对距离改善负责仍不明确，塑形信号越复杂，actor advantage 的方差越难控制。

短程消融见图 \ref{fig:shortablation}。其中 deadlock penalty=0.10 在 5M 内曲线较好，但该现象尚未经过 20M 长跑验证；同时更大的 deadlock penalty 容易压制探索。因此最终报告主结论仍采用更稳的 \code{nodl-nopush}：保留正向势函数，尽量少引入负向规则惩罚。

\begin{figure*}[t]

\caption{5M 短程 reward/model ablation。deadlock 0.10 的短程表现较好，但长程稳定性尚未充分验证；v2 CNN reward 可能带来高 shaped reward，但成功率不稳定。}

\subsection{换模型：CNN 与 GRU 均不是万能提升}
我们比较了 MLP、CNN 和 GRU。MLP 对符号化展平输入表现稳定，是 7x7 主线。CNN 理论上具备局部空间归纳偏置，但在小地图、稀疏回报、通道较少的设置下没有稳定超过 MLP。13x13 random padding 中 CNN 曾出现短暂的 eval success 和 reward 峰值，但持续性不足，说明卷积有助于处理平移，但仅靠 CNN 不能解决探索和信用分配。

GRU 的作用更微妙。对于 turn-based 环境，最近执行动作和当前轮到谁可能帮助识别队友意图；但棋盘状态本身已经记录位置和箱子变化，长历史并非必要。7x7 长跑中，无历史或1步历史 GRU 与 MLP 接近，8步历史后期变差。我们的结论是：动作历史可作为少量辅助输入，但不应替代清晰的状态建模和 reward 设计。

\subsection{真 mix 和 13x13 padding：泛化是主要瓶颈}
图 \ref{fig:random13} 与图 \ref{fig:topleft13} 展示了 13x13 random padding 和 top-left padding 的结果。与 7x7 相比，两者都困难很多：random padding 下多数模型 eval success 长期低于 0.05，CNN 有短暂峰值但不稳；top-left padding 的早期 5M 曲线也基本停留在低成功率区间。图中虚线标记 20M 计划 pretrain/mix 分界，但部分 13x13 结果目前只完成早期阶段，因此更多反映``泛化设置确实更难''，不能简单断言最终上限。

\subsection{HASAC 与 HAHyPO}
HASAC 的训练与评估链路已经打通，最小验证脚本位于 \path{HARL/scripts/verify_sokoban_setup.py}。但在当前离散、稀疏、强协作任务上，off-policy actor-critic（SAC 系列思想见~\citealp{haarnoja2018soft}）更依赖 replay buffer 中的有效推箱样本，早期探索不足时不如 HAPPO 稳定。因此我们将 HASAC 保留为可运行基线入口，主结果使用 HAPPO。

HAHyPO 是我们新增的算法探索：它不改变 critic 的团队回报目标，只修改 actor advantage。该设计适合进一步验证``轨迹级排序信号''能否帮助稀疏合作任务。目前尚无足够完整的长跑曲线支撑其优于 HAPPO，因此报告结论不把它作为最终最佳算法，而作为后续可复现实验方向。

\section{结论与启发}
综合现有材料，我们得到以下结论：
\begin{enumerate}[leftmargin=1.4em,itemsep=0.25em]
    \item 原始 Sokoban reward 合理但稀疏；HAPPO 在 7x7、2 boxes 下可以学到明显策略，但训练较慢。
    \item 最稳的轻量改进是旧 BFS \code{v1-nodl-nopush}：保留箱子到目标和 agent 到箱子的正向势函数，去掉 deadlock 与 pushability 负项。
    \item 更``正确''的 reverse-push v2 并未带来更高成功率，说明 reward shaping 的尺度、可达性 cap 和信用分配比语义完整性更关键。
    \item 模型结构不是主要瓶颈：CNN 和 GRU 都有局部亮点，但没有稳定替代 MLP+良好 shaping；动作历史以 0--1 步较合适，8步历史容易引入噪声。
    \item 真 v0/v1/v2 mix 与 13x13 padding 是质变难度。后续若追求泛化，应优先使用更强的空间归纳偏置、课程学习和地图级 curriculum，而不是继续单点调 reward。
    \item 信用分配仍是核心问题。turn-based active mask 是必要基础；actor-side credit 和 HAHyPO 值得继续做长跑对照，但目前还不能替代 HAPPO 主线结论。
\end{enumerate}
